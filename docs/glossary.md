# 用語集

チュートリアルに登場する用語のリファレンスです。左列は初出の章。

## Terraform 用語

| 章 | 用語 | 意味 |
|---|---|---|
| 00 | **IaC** (Infrastructure as Code) | インフラ構成をコードで宣言し、ツールで実環境に反映するアプローチ |
| 00 | **宣言的** | 手順ではなく「あるべき姿」を書くスタイル。差分の計算はツールに任せる |
| 00 | **provider** | Terraform と各サービス(Snowflake, AWS など)をつなぐプラグイン |
| 02 | **HCL** | Terraform の設定言語。`.tf` ファイルに書く |
| 02 | **resource** | Terraform の管理対象1つを宣言するブロック |
| 02 | **state** | Terraform が「自分の管理下のリソースの現状」を記録したファイル。差分計算の基準 |
| 03 | **init** | 作業ディレクトリの初期化。provider のダウンロードなど |
| 03 | **plan** | 差分の計算と実行計画の表示。環境は変更しない |
| 03 | **apply** | 実行計画の適用。実際に環境を変更する |
| 03 | **destroy** | 管理下のリソースをすべて削除する |
| 03 | **ドリフト** | コード(と state)と実環境のずれ。手作業変更などで発生する |
| 03 | **ロックファイル** (`.terraform.lock.hcl`) | provider バージョンを固定するファイル。コミットする |
| 04 | **リソース間参照** | `リソースタイプ.名前.属性` の形で他リソースの値を参照する仕組み。依存順序も決まる |
| 05 | **variable** | 外部から値を受け取る入力変数。`var.名前` で参照 |
| 05 | **tfvars** | 変数値をまとめたファイル。`-var-file=` で指定 |
| 05 | **locals** | コード内の共通値の定義。`local.名前` で参照 |
| 05 | **output** | apply 結果から値を取り出して表示・連携する仕組み |
| 06 | **backend** | state の保存先設定。ローカル / S3 / Terraform Cloud など |
| 06 | **リモートバックエンド** | state をクラウドに置く構成。チーム開発の前提 |
| 06 | **import** | 手作業で作った既存リソースを Terraform 管理下に取り込む操作 |
| 07 | **module** | resource の集まりを部品化し、変数付きで再利用可能にしたもの |

## Snowflake 用語

| 章 | 用語 | 意味 |
|---|---|---|
| 01 | **Snowsight** | Snowflake の Web 管理画面 |
| 01 | **アカウント識別子** | `組織名-アカウント名` 形式の接続先識別子 |
| 01 | **キーペア認証** | 公開鍵・秘密鍵によるプログラム向け認証方式。パスワードより安全 |
| 01 | **ACCOUNTADMIN** | 最上位の管理ロール。実運用では常用しない |
| 04 | **ウェアハウス** | クエリを実行する計算リソース。稼働時間で課金される |
| 04 | **auto_suspend** | アイドル時にウェアハウスを自動停止する設定。コスト対策の基本 |
| 04 | **ロール** | 権限の束。ユーザーはロール経由で権限を得る |
| 04 | **グラント** | ロールへの権限付与。「DB → スキーマ → オブジェクト」の3段構造 |
| 04 | **future グラント** | 将来作成されるオブジェクトにも自動で権限を付与する仕組み |
| 07 | **SYSADMIN / SECURITYADMIN** | オブジェクト管理 / 権限管理を分担する標準ロール |
| 07 | **機能ロール / アクセスロール** | 人に付ける役割ロールと、オブジェクト単位の権限ロールの2階層設計パターン |
